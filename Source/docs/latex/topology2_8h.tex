\doxysubsection{Detailed Description}
Phase-\/2 topology extensions\+: deterministic hash, monotonic versioning, and named add/update/remove wrappers. 

Spec references\+: AERN-\/2025 Rev. 1a §5–6, Phase-\/2 tasks 2e–2f.

This header adds three categories of new functionality to the base topology\+:


\begin{DoxyEnumerate}
\item {\bfseries{Canonical hash}} (2e)\+: {\ttfamily \doxylink{topology2_8h_ac1abf2bd37fcad1a25566d81d80e1996}{aern\+\_\+topology\+\_\+hash()}} serialises the topology list in serial-\/number order, then computes SHAKE-\/256 over the result, yielding a 32-\/byte digest that is identical across all nodes with the same logical topology.
\item {\bfseries{Monotonic version}} (2f)\+: a {\ttfamily uint64\+\_\+t version} field is embedded in {\ttfamily \doxylink{structaern__topology__versioned__state}{aern\+\_\+topology\+\_\+versioned\+\_\+state}}, which wraps {\ttfamily \doxylink{structaern__topology__list__state}{aern\+\_\+topology\+\_\+list\+\_\+state}}. Any incoming update MUST carry a strictly greater version; otherwise it is rejected with {\ttfamily aern\+\_\+protocol\+\_\+error\+\_\+invalid\+\_\+request}. {\ttfamily \doxylink{topology2_8h_a8a1d167b59105263024b251f93d3eceb}{aern\+\_\+topology\+\_\+increment\+\_\+version()}} atomically increments and returns the new version.
\item {\bfseries{Named wrappers}} (2e)\+: {\ttfamily aern\+\_\+topology\+\_\+add}, {\ttfamily aern\+\_\+topology\+\_\+update}, and {\ttfamily aern\+\_\+topology\+\_\+remove} provide semantically clear entry points that map to the correct underlying list operations. 
\end{DoxyEnumerate}
