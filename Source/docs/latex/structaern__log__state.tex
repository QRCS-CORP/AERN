\doxysubsection{Detailed Description}
State for the ADC encrypted log. 

The log file is encrypted at rest with RCS using a 64-\/byte seed derived from the admin password\+: seed = SHAKE-\/256(password \texorpdfstring{$\vert$}{|}\texorpdfstring{$\vert$}{|} salt)\mbox{[}0..63\mbox{]}

The salt (AERN\+\_\+\+LOG\+\_\+\+SALT\+\_\+\+SIZE = 32 bytes) is stored as a plaintext file alongside the log. The key is never stored; it is re-\/derived on every aern\+\_\+log\+\_\+open() call.

Each log entry is fixed-\/width (AERN\+\_\+\+LOG\+\_\+\+ENTRY\+\_\+\+SIZE bytes) in plaintext, formatted as\+: \texorpdfstring{$<$}{<}ISO-\/8601 UTC timestamp 32B\texorpdfstring{$>$}{>} \texorpdfstring{$<$}{<}device\+\_\+name 32B\texorpdfstring{$>$}{>} \texorpdfstring{$<$}{<}hex\+\_\+error 10B\texorpdfstring{$>$}{>} \texorpdfstring{$<$}{<}error\+\_\+str 64B\texorpdfstring{$>$}{>}~\newline
 No client IP addresses, session IDs, or traffic metadata are written.
