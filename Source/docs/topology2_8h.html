Detailed Description

Phase-2 topology extensions: deterministic hash, monotonic versioning, and named add/update/remove wrappers.

Spec references: AERN-2025 Rev. 1a §5–6, Phase-2 tasks 2e–2f.

This header adds three categories of new functionality to the base topology:

  1. Canonical hash (2e): aern_topology_hash() serialises the topology list in serial-number order, then computes SHAKE-256 over the result, yielding a 32-byte digest that is identical across all nodes with the same logical topology.
  2. Monotonic version (2f): a uint64_t version field is embedded in aern_topology_versioned_state, which wraps aern_topology_list_state. Any incoming update MUST carry a strictly greater version; otherwise it is rejected with aern_protocol_error_invalid_request. aern_topology_increment_version() atomically increments and returns the new version.
  3. Named wrappers (2e): aern_topology_add, aern_topology_update, and aern_topology_remove provide semantically clear entry points that map to the correct underlying list operations.

AERN_EXPORT_API void aern_topology_hash ( const aern_topology_list_state * list,
uint8_t hash_out[AERN_CERTIFICATE_HASH_SIZE] )

Compute the canonical 32-byte SHAKE-256 hash of the topology list.

Nodes are serialised in ascending order of their 16-byte serial numbers (lexicographic comparison) so that all peers with the same logical topology produce an identical hash regardless of insertion order.

Parameters
list[in] The topology list to hash (read-only).
hash_out[out] Output buffer; exactly AERN_CERTIFICATE_HASH_SIZE (32) bytes.

Compute the canonical 32-byte SHAKE-256 hash of the topology list.

This function copies the serialized topology, sorts the copy by certificate serial number, and computes a SHAKE-256 digest over the canonical serialized form.

Parameters
list[const aern_topology_list_state*] The topology list state.
hash[uint8_t*] The output hash buffer; must be AERN_CERTIFICATE_HASH_SIZE bytes.
