Detailed Description

Phase-2 network protocol corrections and hardening.

Spec: AERN-2025 Rev. 1a, §5–6, Phase-2 tasks 2a–2d.

This header exposes:

2a ARS-side remote_signing_response — the corrected handler that explicitly verifies the ADC message signature, the timestamp, and the certificate format before signing, then sends the signed cert back.

2b register_update hardening — topology version-number propagation. The ADC stamps the outgoing topology with the current version; the device verifies incoming_version > current_version before accepting.

2c Client join (aern_network_join_request / response) — full hash-or-full topology logic. Maps to existing register_update_request/response but adds the topology-hash acknowledgement short-path.

2d Convergence update (aern_network_converge_update) — the ADC applies an APS-provided updated node entry after a mismatch is detected.

All functions in this file are additive wrappers or corrections; they do not replace the Phase-1 implementations in network.c (which handle the wire protocol correctly at the packet level).

AERN_EXPORT_API aern_protocol_errors aern_ars_remote_signing_handle ( aern_ars_signing_state * state,
const aern_network_packet * packetin )

ARS-side handler: verify ADC signature, sign the certificate, respond.

Implements the full spec §2a protocol:

  1. Validate header flag, sequence, msglen.
  2. Verify ADC message signature against cached ADC verification key.
  3. Verify the sub-header timestamp ≤ AERN_PACKET_TIME_THRESHOLD.
  4. Deserialize the certificate to be signed.
  5. Hash the cert fields: h = SHA3-256 over child cert fields.
  6. Sign: ars_csig = ML-DSA-Sign(ars_signing_key, h), store in cert csig.
  7. Serialize the signed cert.
  8. Compute: resp_hash = SHA3-256(signed_cert || timestamp_bytes).
  9. Sign response: resp_sig = ML-DSA-Sign(ars_signing_key, resp_hash).

Send: { subheader || signed_cert || resp_sig }.

  1. Zero all intermediate material before returning.
Parameters
state[in,out] ARS signing state.
packetin[in] Incoming remote-signing-request packet.
Returns
Returns aern_protocol_error_none on success.

ADC processes an APS-provided convergence update.

Implements Phase-2 task 2d (ADC convergence resolution):

  1. Verify APS certificate and message signature.
  2. Verify timestamp.
  3. Update topology entry (remove old, insert new).
  4. Increment topology version monotonically. If verification fails, calls aern_network_revoke_broadcast for that node.

AERN_EXPORT_API aern_protocol_errors aern_network_join_request ( aern_network_join_request_state * state)

Client domain-join request.

Implements Phase-2 task 2c (client side):

  1. Serialize: msg = { client_cert || topology_hash || timestamp }.
  2. Sign: msg_sig = ML-DSA-Sign(client_key, SHA3-256(msg)).
  3. Send { msg || msg_sig } to ADC.
  4. Receive response from ADC.
  5. Verify ADC cert root signature and response signature.
  6. If full topology received: flush cache, store new list, update version.
  7. If hash-only: confirm cache is current.

AERN_EXPORT_API aern_protocol_errors aern_network_join_response ( aern_network_join_response_state * state,
const aern_network_packet * packetin )

ADC-side join response handler.

Implements Phase-2 task 2c (ADC side):

  1. Verify client cert root signature.
  2. Verify message signature and timestamp.
  3. Compute current topology hash.
  4. If client topology_hash matches current: send hash-acknowledgement only.
  5. If mismatch: serialize full APS topology, sign, send.
  6. resp_sig covers: adc_cert || topology_or_hash || timestamp.
  7. Increment topology version if client was newly added.
Parameters
state[in,out] Join response state.
packetin[in] Incoming join-request packet.
Returns

ADC-side join response handler.
